132  /*LINKED LIST- 
133  STORES ELEMENT IN DISCONTINUOUS MANNER 
134  ELEMENTS PRESENT IN LINKEDLIST IS CALLED A NODE; 
135  WHERE PREVIOUSLY STORED ELEMENT POINTS TO LOCATION WHERE NEEXT ELEMENT IS STORED; 
136  USED TO PREVENT MEMORY WASTAGE; 
137  LAST ELEMENT IS AWARE THAT THERE IS NO ELEMENT FURTHER; 
138  * RULE FOR CREATING LINKEDLIST; 
139  LinkedList<String> Students = new LinkedList<>(); 
140   
141   
142  >>>LinkedList has all the functionalities of arraylist; 
143   
144  ---for Inserting values in linked list 
145   Students.add(1,4) at index 1 ; value 4 will be inserted and other values indexes change significantly; 
146   
147   
148   -- in arraylist we can sort elements directly but in linkedList we have to manually do it 
149   using algorithms like merge sort; 
150   
151   
152   -- removing an element in LinkedList 
153   arr.remove(2); 
154   whichever element will be existing at index 2 will be  removed 
155   
156  * */

243  *                     LINKED-HASHMAP 
244  * THIS DATA STRUCTURE IS USED TO STORE KEY-VALUE PAIR; 
245  * EXAMPLE 
246  * VALUE OF 
247  *          XYZ -90 //VALUE ASSOCIATED WITH 90 IS XYZ; 
248  *          ABC- 56 
249  *          FGH- 67 
250  * 
251  * MORE SPECIFICALLY -- {1,2,3,1,2,3,1,2,3,1,2,3} 
252  * 
253  * HERE INDEXES OF THESE VALUES CAN ACT AS KEYS AND 
254  * VALUES ITSELF WILL BE VALUES 
255  * TOGETHER THEY WILL FORM KEY-VALUE PAIRS; 
256  SYNTAX           LinkedHashMap<datatype1, datatype2>= new Linkedhashmap<>(); 
257  * datatype1 = key   HEYS ARE UNIQUE IN NATURE; 
258  * datatype2= value 
259  * 
260  * for adding or inserting elements in data structure we use put keyword; 
261  * Numbers.put("abc",89); 
262  * since keys are unique  therefore if we insert same key twice then updated value will be visible; 
263  * order of elements--- the order in wich elements are inserted; 
264  * for calling a value; 
265   */
